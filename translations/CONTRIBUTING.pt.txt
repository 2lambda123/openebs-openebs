Existem algumas diretrizes simples que você precisa seguir antes de fornecer seus hacks.

## Levantando questões

Ao levantar questões, especifique o seguinte:

- Os detalhes da configuração devem ser preenchidos conforme especificado no modelo de problema para que o revisor verifique.
- Um cenário em que o problema ocorreu (com detalhes sobre como reproduzi-lo).
- Erros e mensagens de log que são exibidos pelo software.
- Quaisquer outros detalhes que possam ser úteis.

## Resolva problemas existentes

Vá para [problemas] (https://github.com/openebs/openebs/issues) para encontrar problemas em que a ajuda dos contribuidores é necessária. Consulte nosso [guia da lista de rótulos] (./ contrib / labels-of-issues.md) para ajudá-lo a encontrar problemas que podem ser resolvidos mais rapidamente.

Uma pessoa que deseja contribuir pode resolver um problema reivindicando-o como um comentário / atribuir seu ID do GitHub a ele. No caso de não haver PR ou atualização em andamento por uma semana sobre o referido problema, então o problema é reaberto para qualquer pessoa retomar novamente. Precisamos considerar problemas / regressões de alta prioridade em que o tempo de resposta deve ser de um dia ou mais.

### Assine seu trabalho

Usamos o Certificado de Origem do Desenvolvedor (DCO) como uma proteção adicional para o projeto OpenEBS. Este é um mecanismo bem estabelecido e amplamente utilizado para garantir que os contribuintes tenham confirmado seu direito de licenciar suas contribuições sob a licença do projeto. Leia [certificado-de-origem do desenvolvedor] (./ contribuir / certificado-de-origem do desenvolvedor).

Se você puder certificá-lo, basta adicionar uma linha a cada mensagem git commit:


`` `
  Assinado por: Random J Developer <random@developer.example.org>
`` `

ou use o comando `git commit -s -m" commit message comes here "` para terminar seus commits.

Use seu nome verdadeiro (desculpe, sem pseudônimos ou contribuições anônimas). Se você definir suas configurações git ʻuser.name` e ʻuser.email`, poderá assinar seu commit automaticamente com `git commit -s`. Você também pode usar git [aliases] (https://git-scm.com/book/en/v2/Git-Basics-Git-Aliases) como `git config --global alias.ci 'commit -s'`. Agora você pode fazer um commit com `git ci` e o commit será assinado.
